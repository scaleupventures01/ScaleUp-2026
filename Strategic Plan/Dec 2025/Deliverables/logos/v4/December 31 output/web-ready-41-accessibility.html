ScaleUp Ventures Brand Standards

Typography Standards

Typography is the foundation of accessibility for our 55+ audience. Clear, readable text reduces eye strain and improves comprehension. These standards apply to all ScaleUp Ventures digital and print materials.

1.1 Minimum Font Sizes

Body Text: 18px Minimum
All body copy must be at least 18px (1.125rem). This exceeds the standard 16px recommendation and accounts for reading on various devices and lighting conditions.
  • Desktop body text: 18px-20px
  • Mobile body text: 18px minimum (do not reduce)
  • Long-form content: 20px recommended
  • Captions and fine print: 16px absolute minimum
Headlines: 24px+ Minimum
Headlines must create clear visual hierarchy while remaining readable:
  • H1 (Page titles): 42px-56px
  • H2 (Section headers): 32px-42px
  • H3 (Subsection headers): 24px-32px
  • H4 (Minor headers): 20px-24px
Element Minimum Size Recommended CSS
Body text 18px 18-20px font-size: 1.125rem
H1 42px 48-56px font-size: 2.625rem
H2 32px 36-42px font-size: 2rem
H3 24px 26-32px font-size: 1.5rem
H4 20px 20-24px font-size: 1.25rem
Captions 16px 16-18px font-size: 1rem
Button text 16px 18px font-size: 1.125rem
Form labels 16px 18px font-size: 1.125rem

1.2 Line Height & Spacing

Line Height: 1.6 Minimum for Body Text
Generous line spacing improves readability and reduces visual fatigue:
  • Body text: 1.6-1.8 line height
  • Headlines: 1.2-1.3 line height
  • Lists: 1.5 line height within items
  • Paragraph spacing: 24px between paragraphs
Letter Spacing
  • Body text: Normal (0)
  • Headlines in Crimson Pro: 0 to -0.02em
  • All caps text: +0.05em minimum
  • Never use condensed letter spacing in body copy

1.3 Font Family Specifications

Crimson Pro (Headlines)
  • Use for: Page titles, section headers, hero text
  • Weights: 500 (Medium), 600 (SemiBold), 700 (Bold)
  • Never use: Light (300) or Regular (400) for headlines
  • Fallback: Georgia, "Times New Roman", serif
DM Sans (Body Copy)
  • Use for: All body text, buttons, forms, navigation
  • Weights: 400 (Regular), 500 (Medium), 600 (SemiBold), 700 (Bold)
  • Never use: Light weights for body text
  • Fallback: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif
/* ScaleUp Typography CSS */ body { font-family: 'DM Sans', -apple-system, BlinkMacSystemFont, sans-serif; font-size: 18px; line-height: 1.7; font-weight: 400; } h1, h2, h3 { font-family: 'Crimson Pro', Georgia, serif; font-weight: 600; line-height: 1.2; } h1 { font-size: clamp(42px, 5vw, 56px); } h2 { font-size: clamp(32px, 4vw, 42px); } h3 { font-size: clamp(24px, 3vw, 32px); }

1.4 Font Weight Guidelines

Important: Avoid Thin Weights
Never use font weights below 400 for any text. Light and thin weights (100-300) significantly reduce readability for users with vision changes common in the 55+ demographic.
Weight Use Case Approved
300 (Light) Never use No
400 (Regular) Body text, form fields Yes
500 (Medium) Subheadings, emphasized text Yes
600 (SemiBold) Headlines, buttons, labels Yes
700 (Bold) Primary headlines, CTAs Yes

1.5 Text Formatting Rules

Avoid All Caps
All-caps text is harder to read because it removes word shape recognition:
  • Never use ALL CAPS for sentences or paragraphs
  • Limit to: Button text (2-3 words max), Labels, Short navigation items
  • When using all caps, increase letter spacing by 0.05em
Italics Usage
  • Use sparingly for emphasis or citations
  • Never italicize entire paragraphs
  • Avoid for navigation or UI elements
Underlines
  • Reserve underlines exclusively for links
  • Never underline for emphasis (use bold instead)
  • Link underlines should be visible and consistent

Color & Contrast

Color contrast directly impacts readability. Our 55+ audience may have reduced contrast sensitivity, making proper color ratios essential. All color combinations must meet WCAG AA standards (4.5:1 for normal text, 3:1 for large text).

2.1 Primary Color Specifications

2.2 Contrast Ratios

Combination Contrast Ratio WCAG Rating Use For
Navy on White 16.4:1 AAA Body text, headlines
White on Navy 16.4:1 AAA Inverted sections, buttons
Gray 600 on White 5.9:1 AA Secondary text, captions
Gold on Navy 7.2:1 AAA Accents on dark backgrounds
Navy on Gold 7.2:1 AAA Text on gold highlights
Gold on White 2.3:1 Fail NEVER use for text

2.3 Approved Color Combinations

Navy text on White - Primary body text combination 16.4:1 AAA
White text on Navy - Headers, hero sections, buttons 16.4:1 AAA
Gold text on Navy - Accents, highlights, labels 7.2:1 AAA
Gray 600 on White - Secondary text, descriptions 5.9:1 AA

2.4 Prohibited Color Combinations

Gold text on White - Never use for any text 2.3:1 Fail
Light gray on light gray - Insufficient contrast 1.9:1 Fail
Gold Text Warning
Gold (#D4A853) should ONLY be used for text when placed on Navy backgrounds. On white or light backgrounds, gold fails contrast requirements. Use gold only for decorative elements, borders, or icons on light backgrounds.

2.5 Link Colors

Text Links
  • Default state: Navy (#0F1729) with underline
  • Hover state: Gold (#D4A853) underline
  • Visited state: Navy at 80% opacity or purple (#5b21b6)
  • Focus state: 3px gold outline, 2px offset
Links on Dark Backgrounds
  • Default: White with underline
  • Hover: Gold (#D4A853)
  • Focus: 3px white outline

2.6 Button States

Button Type Default Hover Focus Disabled
Primary Navy bg, white text Darker navy (#0a1018) 3px gold outline 50% opacity
Secondary White bg, navy border Gray-100 bg 3px gold outline 50% opacity
Gold Accent Gold bg, navy text Darker gold (#c49743) 3px navy outline 50% opacity

2.7 Color Independence

Never Rely on Color Alone
Always provide secondary indicators alongside color:
  • Error states: Red color + icon + descriptive text
  • Success states: Green color + checkmark icon + confirmation text
  • Required fields: Asterisk (*) + "(required)" text
  • Links: Color change + underline
  • Charts: Color + patterns + labels

Layout & Spacing

Generous whitespace and appropriate content widths reduce cognitive load and improve focus. These layout standards ensure comfortable reading and navigation.

3.1 White Space Requirements

Minimum Margins & Padding
  • Page margins: 24px minimum on mobile, 48px+ on desktop
  • Section padding: 48px-64px vertical spacing between sections
  • Card padding: 24px minimum internal padding
  • Button padding: 16px vertical, 24px horizontal minimum
8px Grid System
All spacing should be multiples of 8px:
  • 8px - Tight spacing (between related elements)
  • 16px - Standard element spacing
  • 24px - Card padding, form field spacing
  • 32px - Section sub-divisions
  • 48px - Major section spacing
  • 64px - Page section divisions

3.2 Content Width Limits

Content Type Max Width Optimal Width
Body text lines 75 characters 60-70 characters
Main content container 900px 800px
Full-width sections 1400px 1200px
Form containers 600px 500px
Card grids 1200px 1000px
Line Length Matters
Lines longer than 75 characters require excessive eye movement and make it difficult to track to the next line. This is especially problematic for readers with reduced visual acuity.

3.3 Mobile Considerations

Touch Target Sizes
All interactive elements must be easily tappable:
  • Minimum touch target: 44px x 44px (48px x 48px recommended)
  • Spacing between targets: 8px minimum
  • Links in text: Generous padding, clear tap areas
Mobile Typography
  • Never reduce body text below 18px on mobile
  • Headlines scale down proportionally but never below 24px for H3
  • Maintain 1.6+ line height on all devices
  • Increase tap area padding on links
Mobile Navigation
  • Hamburger menu icon: 48px x 48px minimum
  • Menu items: Full-width, 56px height minimum
  • Close button: Prominent, 48px tap target
  • Fixed header: Maximum 64px height to preserve content

3.4 Responsive Breakpoints

Breakpoint Width Considerations
Mobile 320px - 639px Single column, stacked layouts, 24px margins
Tablet 640px - 1023px 2-column grids, 32px margins
Desktop 1024px - 1399px Multi-column, 48px margins
Large Desktop 1400px+ Max-width containers, centered content

3.5 Visual Hierarchy

Clear Content Structure
  • One primary action per viewport
  • Clear visual grouping with whitespace
  • Consistent heading hierarchy (never skip levels)
  • Important content placed top-left for natural reading flow
  • Progressive disclosure for complex information

Form Design

Forms are critical conversion points. Our 55+ audience needs clear labels, generous field sizes, and helpful error messaging to complete forms successfully.

4.1 Field Size Requirements

Input Field Dimensions
  • Minimum height: 48px (56px recommended)
  • Internal padding: 16px horizontal, 12px vertical
  • Font size in fields: 18px minimum
  • Border width: 2px (visible boundaries)
  • Border radius: 8px for softer appearance
Accessible Form Field Example

We will use this to send your consultation details.

4.2 Label Placement

Labels Above Fields
Always place labels above input fields, never inside (as placeholders) or to the side:
  • Label position: Directly above the field
  • Label spacing: 8px below label, before field
  • Label weight: 600 (SemiBold) for visibility
  • Required indicator: Red asterisk + "(required)" for screen readers
Do This

Labels above fields are easier to scan

Avoid This

Placeholder-only labels disappear on focus

4.3 Error Messaging

Error Display Requirements
  • Position: Directly below the field with error
  • Color: Red (#dc2626) text + red border on field
  • Icon: Error icon before message text
  • Message: Specific, actionable instructions
  • Focus: Auto-focus to first field with error
Error State Example

Please enter a complete email address (example: john@company.com)

4.4 Form Accessibility Features

Required Implementation
  • All form fields must have visible labels (no placeholder-only)
  • Labels must be programmatically linked to inputs (for/id or aria-labelledby)
  • Error messages must be linked with aria-describedby
  • Form submission status announced to screen readers
  • Tab order must follow visual order
Helper Text
  • Provide format hints for phone numbers, dates
  • Explain why information is needed
  • Show character counts for limited fields
  • Indicate password requirements upfront

4.5 Button Specifications

Images & Graphics

Visual content must be accessible to all users, including those using screen readers or experiencing reduced vision. Proper alt text and sizing ensure images enhance rather than hinder the user experience.

5.1 Alt Text Requirements

Descriptive Alt Text Guidelines
  • Maximum length: 125 characters (screen readers may truncate longer)
  • Describe the content and function, not appearance
  • Include text visible in the image
  • Avoid "Image of..." or "Picture of..." prefixes
  • For decorative images, use empty alt="" (not missing alt)
Good Alt Text

Image: Business owner shaking hands

alt="Jennifer Miller, owner of Miller Staffing, celebrates closing a $2M client"

Poor Alt Text

Image: Business owner shaking hands

alt="Image of two people shaking hands"

5.2 Image Categories & Alt Text

Image Type Alt Text Approach Example
Informative Describe content and purpose "Chart showing 47% revenue growth over 18 months"
Decorative Empty alt="" Background patterns, divider lines
Functional (links) Describe destination/action "Download the complete case study PDF"
Complex (charts) Brief alt + detailed description nearby "Revenue comparison chart. Details in table below."
Logos Company name "ScaleUp Ventures"

5.3 Icon Sizing

Minimum Icon Dimensions
  • Standalone icons: 24px x 24px minimum, 32px recommended
  • Icons in buttons: 20px x 20px minimum, aligned with text
  • Navigation icons: 24px x 24px minimum
  • Social media icons: 32px x 32px minimum (tap targets 44px)
  • Status indicators: 16px minimum, 20px recommended
Icon Accessibility
  • Icons with meaning: Include aria-label or visible text
  • Decorative icons: Use aria-hidden="true"
  • Icon buttons: Always include accessible name
  • Icon color: Must meet 3:1 contrast against background

5.4 Chart Readability

Chart Design for 55+ Audience
  • Use high-contrast color combinations
  • Label data points directly (avoid relying on legends)
  • Font size in charts: 14px minimum, 16px recommended
  • Line thickness: 3px minimum for data lines
  • Data point markers: 8px minimum diameter
Chart Accessibility Requirements
  • Provide data table alternative for all charts
  • Use patterns in addition to colors
  • Include summary text describing trends
  • Avoid 3D effects that distort data
  • Limit pie charts to 5 segments maximum

5.5 Logo Usage

ScaleUp Ventures Logo Specifications
  • Minimum width: 120px (digital), 1 inch (print)
  • Clear space: Equal to height of "S" in ScaleUp on all sides
  • Alt text: "ScaleUp Ventures" (not "ScaleUp Ventures logo")
  • Link home when used as primary logo

5.6 Image Resolution

Use Case Minimum Resolution Format
Hero images 1920px wide, 2x for retina WebP with JPG fallback
Thumbnails 400px wide, 2x for retina WebP with JPG fallback
Team photos 600px wide minimum WebP with JPG fallback
Icons SVG preferred SVG or PNG at 2x
Logos SVG preferred SVG with PNG fallback

10-Point Accessibility Testing Checklist

Use this checklist before publishing any ScaleUp Ventures material. All items must pass before launch.

1. Text Size & Readability Body text is 18px+, headlines are 24px+, line height is 1.6+, and line length is under 75 characters.
2. Color Contrast All text meets WCAG AA standards (4.5:1 for normal text, 3:1 for large text). Test with WebAIM Contrast Checker.
3. Keyboard Navigation All interactive elements are reachable via Tab key. Focus indicators are visible (3px outline). Tab order follows visual order.
4. Touch Targets All buttons and links have 44x44px minimum tap targets. Spacing between targets is at least 8px.
5. Form Labels All form fields have visible labels above them. Labels are linked to inputs. Required fields are clearly marked.
6. Error Messages Error messages are specific and actionable. Errors use color + icon + text. Focus moves to first error field.
7. Image Alt Text All meaningful images have descriptive alt text. Decorative images have empty alt="". No "Image of..." prefixes.
8. Heading Structure Headings follow logical order (H1 > H2 > H3). No heading levels are skipped. Only one H1 per page.
9. Link Clarity Link text describes destination (no "click here"). Links are underlined. Visited links are distinguishable.
10. Mobile Responsiveness Content is readable without horizontal scrolling. Text doesn't shrink below 18px. Primary CTA is visible without scrolling.

Testing Tools

Recommended Accessibility Testing Tools

Implementation Checklist

Use this implementation checklist when creating new ScaleUp Ventures materials:

Before Starting

Load brand fonts Include Crimson Pro (500, 600, 700) and DM Sans (400, 500, 600, 700) from Google Fonts
Set CSS variables Define --navy: #0F1729 and --gold: #D4A853 for consistent usage
Set base typography font-size: 18px, line-height: 1.7 on body element

During Development

Use semantic HTML Proper heading hierarchy, landmarks, lists, and form elements
Add alt text as you go Write alt text when adding each image, not after
Test keyboard navigation Tab through each section as you build it
Check mobile view Test responsiveness at 375px width frequently

Before Publishing

Run WAVE or axe Fix all errors, review all warnings
Test with screen reader Listen to full page announcement, verify logical flow
Complete 10-point checklist All items must pass before publishing
