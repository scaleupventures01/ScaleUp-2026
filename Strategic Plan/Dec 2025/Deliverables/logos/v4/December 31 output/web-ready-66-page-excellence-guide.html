Design Foundations & Color System

Why This Matters

ScaleUp Ventures' website must immediately communicate certainty, professionalism, and trustworthiness. The design should feel like a patient observer - watchful, measured, decisive. NOT generic, NOT flashy, NOT typical agency aesthetic.

CRITICAL: Avoid Generic AI Aesthetics

  • NO gradient meshes or aurora backgrounds
  • NO generic blue/purple tech color schemes
  • NO stock photos of people pointing at screens
  • NO "innovative, cutting-edge, revolutionary" language
  • NO complex animations that distract from content

Recommended Color System

/* COLOR SYSTEM - Use these exact values */ :root { /* Primary */ --primary-dark: #0F1729; /* Deep navy - trust, professionalism */ --accent-gold: #D4A853; /* Warm gold - success, premium */ --cream-bg: #FAF8F5; /* Warm background - NOT generic white */ /* Functional */ --certainty-blue: #2563EB; /* Data, mathematical precision */ --proof-green: #10B981; /* Success states, guarantees */ --warning-orange: #F59E0B; /* Cautions */ --error-red: #EF4444; /* Errors */ /* Text */ --text-primary: #1a1a2e; /* Headlines, body */ --text-secondary: #64748b; /* Supporting copy */ }

Typography Hierarchy

Font Pairing (Distinctive, Not Generic)

  • Headlines: Crimson Pro (serif) - Confident, authoritative, NOT typical tech sans-serif
  • Body: DM Sans - Clean, readable, professional
  • Accent/Data: Fira Code or mono font for numbers and stats
/* TYPOGRAPHY SCALE */ --font-display: 3.5rem; /* 56px - Hero headlines only */ --font-h1: 2.5rem; /* 40px - Page titles */ --font-h2: 1.75rem; /* 28px - Section headers */ --font-h3: 1.25rem; /* 20px - Card titles */ --font-body: 1rem; /* 16px - Body copy */ --font-small: 0.875rem; /* 14px - Captions, metadata */ /* Line heights */ --leading-tight: 1.2; /* Headlines */ --leading-normal: 1.7; /* Body copy */

Visual Identity: The Patient Observer

Attribute Avoid Use Instead
Animations Fast, flashy, bouncing Slow, measured, purposeful
Colors Bright gradients, neon accents Deep navy, warm gold, cream
Imagery Stock photos, generic tech imagery Data visualizations, real results, screenshots
Spacing Cramped, busy layouts Generous whitespace, breathing room
Typography All sans-serif, generic Serif headlines, distinctive hierarchy

Contact Page Recommendations

Frame as Advisory, Not Transactional

Headline Approach

The contact page is for visitors ready to engage. Use Stage 3-4 CTAs and advisory framing.

<!-- CONTACT PAGE HERO --> <section class="contact-hero"> <!-- Advisory framing, not sales --> <h1>See Your Numbers First</h1> <p class="subhead"> This isn't a sales call. It's a reality check. We'll show you the projected ROI—then you decide. </p> <!-- Set expectations --> <div class="what-to-expect"> <h3>What to Expect:</h3> <ul> <li>15-minute call (we respect your time)</li> <li>We'll ask about your current situation</li> <li>You'll get a preliminary ROI projection</li> <li>No pressure. No follow-up spam.</li> </ul> </div> </section>

Form Best Practices

Progressive Disclosure

  • Step 1: Name, Email, Company (required)
  • Step 2: What service are you interested in? (dropdown)
  • Step 3: Brief description of your situation (optional textarea)

Multi-step forms have 25-30% higher completion rates than single long forms.

Primary Action: Calendly Embed

Make booking a call the PRIMARY action. Form is secondary for those who prefer email.

UI/UX Best Practices

Visual Hierarchy

F-Pattern Reading for Homepage

  • Top bar: "Certainty First." badge + navigation
  • First horizontal: Hero headline + CTA
  • Left scan: Section headers down the page
  • Second horizontal: Social proof / results

Mobile Responsiveness

Critical Specifications

  • Min touch target: 48px × 48px
  • Primary breakpoint: 768px
  • Mobile-first development: Start at 375px
  • Font scaling: 16px minimum body text

Accessibility (WCAG 2.1 AA)

Non-Negotiable Requirements

  • Color contrast: 4.5:1 minimum for body text
  • Focus states: Visible keyboard focus on all interactive elements
  • Alt text: Descriptive alt text on all images
  • Form labels: Every input must have an associated label
  • Skip links: "Skip to main content" link for keyboard users

Performance Targets

Metric Target Why It Matters
Largest Contentful Paint (LCP) < 2.5s User perceives page as fast
First Input Delay (FID) < 100ms Interactions feel responsive
Cumulative Layout Shift (CLS) < 0.1 Page doesn't jump around
Time to Interactive (TTI) < 3.8s Page is usable quickly

Trust Signals Placement

Social Proof Hierarchy

  1. Above fold: Key metric or guarantee badge
  2. After hero: Client logos (if available)
  3. Mid-page: Featured testimonial with specific results
  4. Near CTAs: "No commitment. No credit card."
  5. Footer: Certifications, security badges (if applicable)

Content Length Strategy: Desktop vs Mobile

CRITICAL: Why Text Length Matters for Conversion

Too much text = visitors bounce. Too little = not enough information to convert. The goal is scannable content that builds confidence quickly.

  • Desktop: Users read more, tolerate longer content
  • Mobile: Users scan, need bite-sized content, thumb-scrolling

How It Works: The 3-Tier Content Approach

You Are NOT Writing Two Complete Websites

You write one primary version (desktop), then create 5-10 mobile variants for key sections only. CSS handles everything else.

Tier What It Is % of Content How It Works
Tier 1 Same content, different layout 80% CSS handles it (grid, flexbox, font sizes)
Tier 2 Two written versions 15% Desktop-only / Mobile-only classes
Tier 3 Progressive disclosure 5% "Read more" expanders on mobile

Tier 1: Same Content, Responsive Layout (80%)

Most Content Stays the SAME Words

You write it once. CSS changes how it displays on different screen sizes.
